The Automated MLOps Factory

From a blueprint to a globally deployed product. Hover over any step to see its role.

1. Code Push to GitHub

New Blueprint

A developer submits a new design by running `git push` to the central blueprint office, GitHub. This is the trigger.

CI Pipeline (Continuous Integration)

2. Jenkins Pipeline Starts

Start Assembly Line

The Robot Foreman (Jenkins) is notified and starts the entire automated pipeline defined in the `Jenkinsfile`.

3. Ingest & Process Data

Fabricate Parts

Jenkins runs scripts to order raw materials from GCP, then fabricates and prepares all components (feature engineering, scaling).

4. Train & Track Model

Assemble Engine

The engine is assembled and tuned. All performance results and the final engine artifact are logged to the MLflow testing bay.

CD Pipeline (Continuous Deployment)

5. Build Application Image

Final Assembly

The finished engine is placed in the car chassis (Flask app) and the whole car is sealed in a standardized Docker shipping container.

6. Push to GCR

Ship to Showroom

The containerized car is pushed to the Google Container Registry, our secure global showroom for finished products.

7. Deploy to Cloud Run

Go Live!

Jenkins places an order, and Cloud Run pulls the latest car from the showroom and instantly makes it available to the public.
